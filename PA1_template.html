That doesn't matter because in time series analysis it is normally assumed that the observations are evenly distanced in time. So, no further measures are taken to transform the intervals into R standard date or time formats.

Aggregate steps by interval with basic R function aggregate():

stepave <- aggregate(activity$steps, by=list(activity$interval), 
                     FUN=mean, na.rm=TRUE)
colnames(stepave) <- c("interval", "steps.mean")

A bunch of alternatives to do this task can be found at stackoverflow. The following is a time series plot of the 5-minute interval (x-axis) and the average number of steps taken, averaged across all days (y-axis).

aveplot <- ggplot(stepave, aes(x=interval, y=steps.mean)) + geom_line()
aveplot + ylab("average number of steps taken")

The maximum of steps taken, averaged across all days, is:

Imputing missing values

Note that there are a number of days/intervals where there are missing values (coded as NA). The presence of missing days may introduce bias into some calculations or summaries of the data.

Calculate the total number of missing values in the dataset (i.e. the total number of rows with NAs)

This is telling us that there are 2304 NA values in the activity data, all of them in the steps, none in date or interval. Double check this with the complete.cases() function:

nrow(activity) - sum(complete.cases(activity))

## [1] 2304

A valid strategy for filling in all of the missing values in the dataset is to use the mean of available values of steps taken in that 5-minute interval, averaged across all days.

Create a new dataset that is equal to the original dataset but with the missing data filled in. First there will be a data dictionary to access means of steps

mean.dict <- stepave$steps.mean
names(mean.dict) <- stepave$interval
stepmeans <- mean.dict[as.character(activity$interval)]

Copy activity table to replace cell contents of steps in case values are not available and fill in steps values from dictionary.

actfill <- activity
actfill$steps[is.na(actfill$steps)] <- stepmeans[is.na(actfill$steps)]

Double check the result.

sum(is.na(activity))

## [1] 2304

sum(is.na(actfill))

## [1] 0

summary(actfill$steps - activity$steps)

##    Min. 1st Qu.  Median    Mean 3rd Qu.    Max.    NA's 
##       0       0       0       0       0       0    2304

So, all steps data are identical, except where no data were available. No significant data were overwritten, all NA were replaced.

Are there differences in activity patterns between weekdays and weekends?

To split the activities in workdays and weekend use another hint from stackoverflow. This allows an operation on weekday names instead of numbers, and avoids hassles with the localization at the same time. 

weekend <- weekdays(as.Date(4,"1970-01-01",tz="GMT")+5:6)

According to this hint append another column to the activities:

actfill$daygroup <- ifelse(weekdays(actfill$date) %in% weekend, 
                            "weekend", "weekday")

The total number of taken steps in each 5-minute interval will be averaged, and grouped by weekday resp. weekend.

actweekaggr <- aggregate(actfill$steps, by=list(actfill$interval, actfill$daygroup), 
                     FUN=mean, na.rm=TRUE)
colnames(actweekaggr) <- c("interval", "daygroup", "steps")

Eventually the graphs are plotted in a facet grid

g <- ggplot(actweekaggr, aes(x=interval, y=steps)) + geom_line()
g + facet_grid(daygroup ~ .) + ylab("average number of steps taken")

Comparing weekdays and weekend, there is a much higher rate of activity in the weekday morning between 5:00h and about 9:00h. Through the day the activity level at the weekend is about twice as high as at weekdays. After 17:00h there is a nip-and-tuck race until the activity dies off evenly from 21:00h to night in both groups.
